## 🎯 시스템 개요

### 목적

**대형 프로젝트 운영관리의 완전 자동화 및 최적화**

- ✅ **자동 모니터링**: 프로젝트 상태, 파일 변경, 작업 흐름 자동 추적
- ✅ **지능형 보고**: 에이전트 팀 기반 전문가급 분석 및 보고
- ✅ **검증된 데이터 기반**: MCP 활용으로 뇌피셜 없는 정확한 분석
- ✅ **확장 가능한 아키텍처**: 프로젝트 규모에 따라 자동 확장
- ✅ **파일 종속성 제로**: 유저룰스만으로 완전 독립 운영

### 핵심 가치

1. **최소 의존성**: 외부 스크립트나 복잡한 설정 없이 유저룰스만으로 작동
2. **최대 자동화**: 수동 작업 최소화, 지능형 자동 실행
3. **확장성**: 소규모부터 엔터프라이즈급까지 대응
4. **검증 가능성**: 모든 분석과 보고는 검증된 데이터 기반
5. **유지보수성**: 모듈화된 구조로 쉬운 관리 및 확장

### 핵심 컴포넌트

#### 1. 유저룰스 (모니터링 및 판단 계층)

**역할**: AI의 행동 지침 및 자동화 규칙 정의

- **모니터링 규칙**: 파일 변경, 작업 완료, 상태 변화 감지
- **트리거 규칙**: 조건 기반 자동 실행 규칙
- **검색 시스템**: 컨텍스트 기반 유저커맨드 검색 및 매칭
- **에이전트 시스템**: 전문가 팀 기반 분석 및 보고

**특징**:
- 파일 종속성 없음 (독립 모듈)
- 모듈화된 구조 (필요한 모듈만 선택)
- 확장 가능한 설계

#### 2. 유저커맨드 (실행 계층)

**역할**: 실제 작업 수행 및 파일 생성/업데이트

- **시간 관리**: Clock.json 동기화
- **프로젝트 초기화**: 필수 파일 생성
- **로그 생성**: 시계열 기반 로그 관리
- **자동 유지보수**: 백그라운드 모니터링 및 유지보수

**특징**:
- 헤더 주석 기반 검색 가능
- 파일 생성/업데이트 보장
- 백그라운드 실행 지원

#### 3. !AGENT_System/ (데이터 계층)

**역할**: 시계열 기반 데이터 저장 및 관리

- **보고서**: 유저 요청 시 생성되는 전문가 분석 보고서
- **LOG**: 트리거 기반 자동 생성되는 요약 로그
- **시계열 관리**: 파일명에 타임스탬프 포함으로 자동 정렬

**특징**:
- 단일 폴더 구조 (복잡한 하위 폴더 없음)
- 시계열 기반 파일명 (YYYYMMDD_HHMMSS_형식)
- 파일 링크 기반 종속관계 관리

## 🎨 핵심 철학 및 설계 원칙

### 1. Zero Dependency (제로 의존성)

**원칙**: 외부 스크립트나 복잡한 설정 파일에 의존하지 않음

**구현**:
- 모든 규칙은 `.cursorrules` 파일에 포함
- 모든 커맨드는 `project-commands.md`에 포함
- 파일 종속성 없는 독립 모듈 설계

**장점**:
- 이식성: 프로젝트 간 쉽게 이동 가능
- 유지보수: 단일 파일 관리로 간단
- 안정성: 외부 의존성으로 인한 오류 방지

### 2. Context-Based Intelligence (컨텍스트 기반 지능)

**원칙**: 하드코딩된 규칙 대신 컨텍스트 분석 기반 동작

**구현**:
- 헤더 주석 기반 유저커맨드 검색
- 매칭 점수 계산으로 최적 커맨드 선택
- 동적 에이전트 팀 생성

**장점**:
- 유연성: 상황에 맞는 최적의 동작
- 확장성: 새 커맨드 추가 시 자동 통합
- 지능성: AI의 판단 능력 최대 활용

### 3. Verified Data First (검증된 데이터 우선)

**원칙**: 뇌피셜 금지, 검증된 데이터만 사용

**구현**:
- MCP 활용 (Exa Search, Context7, Browser Use)
- 에이전트 팀의 검증된 데이터 기반 분석
- 파일 링크 기반 환각 관리

**장점**:
- 정확성: 신뢰할 수 있는 분석 결과
- 추적 가능성: 데이터 소스 명확
- 검증 가능성: 모든 분석 결과 검증 가능

### 4. Modular Architecture (모듈화 아키텍처)

**원칙**: 독립적인 모듈로 구성, 필요에 따라 선택적 사용

**구현**:
- 6개 독립 모듈 (마스터, 파일관리, 운영관리, 커맨드, 에이전트, 파일링크)
- 각 모듈은 독립적으로 유저룰스에 붙여넣기 가능
- 파일 종속성 없음

**장점**:
- 유연성: 프로젝트에 맞는 모듈만 선택
- 확장성: 새 모듈 추가 용이
- 유지보수: 모듈별 독립 관리

### 5. Time-Series Management (시계열 관리)

**원칙**: 시간 기반 데이터 관리로 자동 정렬 및 추적

**구현**:
- 파일명에 타임스탬프 포함 (YYYYMMDD_HHMMSS)
- 시계열 순서로 자동 정렬
- 시간 기반 트리거 및 분석

**장점**:
- 자동 정렬: 시간 순서로 자동 정리
- 추적 가능: 시간 기반 이력 추적
- 분석 용이: 시간 패턴 분석 가능

## 🚀 대형 프로젝트 운영관리 전략

### 1. 자동 모니터링 시스템

#### 파일 모니터링

**대상**:
- 파일 생성/수정/삭제
- 폴더 구조 변경
- Git 상태 변화

**작동 방식**:
```
파일 변경 감지
  → 모니터링 (관여 없음)
  → 로그 기록 (!AGENT_System/)
  → 상태 업데이트
  → 트리거 조건 확인
  → 필요시 자동 실행
```

**장점**:
- 실시간 프로젝트 상태 파악
- 자동 이력 관리
- 문제 조기 감지

#### 사용자 컨텍스트 모니터링

**대상**:
- 사용자 요청
- 작업 의도
- 진행 상황
- 문제 발생

**작동 방식**:
```
사용자 요청/작업 감지
  → 컨텍스트 분석
  → 맥락 업데이트
  → 관련 에이전트 팀 활성화
  → 전문가급 분석 수행
```

**장점**:
- 사용자 의도 정확 파악
- 맥락 기반 최적 동작
- 지능형 자동화

### 2. 에이전트 팀 시스템

#### 핵심 팀 구조

**총괄 팀 (Chief Team)**
- 역할: 백그라운드 프로젝트 모니터링 및 총괄 관리
- 슈퍼바이저: 최고 (혁신 기술) + 시니어 (검증 데이터)
- 트리거: 파일 변경, 작업 완료, 문제 발생, 주기적(1시간)
- 출력: 상태 요약, 문제 감지, 개선 제안

**PM 팀 (Project Manager Team)**
- 역할: 일정 관리, 진행률 추적, 마일스톤 관리
- 슈퍼바이저: 최고 (최신 방법론) + 시니어 (검증된 사례)
- 트리거: 진행률 변화(10% 이상), 마일스톤, 일정 변경
- 출력: 진행률 보고, 일정 분석, 리스크 평가

**관리자 팀 (Administrator Team)**
- 역할: 전략 수립, 효율성 분석, 리소스 관리
- 슈퍼바이저: 최고 (최신 전략) + 시니어 (검증된 전략)
- 트리거: 효율성 지표 변화, 리소스 사용 변화
- 출력: 전략 제안, 효율성 분석, 최적화 제안

**감사 팀 (Audit Team)**
- 역할: 품질 관리, 표준 준수, 코드 리뷰
- 슈퍼바이저: 최고 (최신 표준) + 시니어 (검증된 표준)
- 트리거: 품질 문제 감지, 표준 위반, 코드 변경
- 출력: 품질 보고, 표준 준수 분석, 개선 제안

#### 동적 팀 생성

**조건**: 전문 분야 작업 필요, 해당 팀 없음

**지원 팀**:
- 코딩 팀: 기술 스택, 패턴, 모범 사례
- 디자인 팀: 디자인 사례, 트렌드, 도구
- UX 팀: UX 연구, 사례, 방법론
- DX 팀: 개발 경험, 도구, 방법론

**작동 방식**:
```
전문 분야 작업 감지
  → 팀 존재 확인
  → 없으면 생성 (최고 + 시니어 슈퍼바이저)
  → MCP 활성화 확인 및 요청
  → 팀 활성화 및 작업 진행
```

### 3. 보고 시스템

#### 보고서 (유저 요청 시만)

**생성 조건**: 사용자 명시적 요청 ("보고", "리포트", "상태", "분석")

**형식**: `!AGENT_System/YYYYMMDD_HHMMSS_보고자_보고명.MD`

**내용**:
- 현재 상태 요약
- 완료된 작업
- 문제점 및 개선 사항
- 다음 단계 제안
- 관련 파일 링크 (맥락관리, 환각관리, 버전관리)

**특징**:
- 전문가급 분석
- 검증된 데이터 기반
- 파일 링크로 추적 가능

#### LOG (트리거 기반 자동)

**생성 조건**:
- 주기적 (1시간)
- 진행률 변화 (10% 이상)
- 마일스톤 달성/미달성
- 문제 발생
- 파일 변경 누적 (10개 이상)

**형식**: `!AGENT_System/YYYYMMDD_HHMMSS_LOG_보고자_요약명.MD`

**내용**:
- 트리거 기간 요약
- 주요 변경사항
- 관련 파일 링크
- 버전 정보

**특징**:
- 자동 생성
- 핵심만 요약
- 시계열 추적 가능

### 4. 검증 데이터 기반 작업

#### MCP 활용

**필수 MCP**:
- **Exa Search**: 논문, Git 저장소, 리서치, 포럼 검색
- **Context7**: 라이브러리 문서, API 문서
- **Browser Use**: 실제 웹사이트 확인 (감사 작업 필수)

**작동 방식**:
```
작업 시작
  → 필수 MCP 목록 확인
  → 활성화 상태 체크
  → 비활성화 시 사용자 요청
  → 활성화 후 MCP 활용
  → 검증된 데이터 수집
  → 데이터 기반 작업 수행
```

**장점**:
- 최신 정보 활용
- 검증된 데이터 기반
- 정확한 분석 결과

- 사용자가 선택적으로 붙여넣기 가능

### 2. 커맨드 확장

#### 새 커맨드 추가

**절차**:
1. `project-commands.md`에 새 함수 추가
2. 헤더 주석 작성 (필수 필드 포함)
3. `cursor_command()` 라우터에 등록
4. 별칭 추가 (선택)

**헤더 주석 형식**:
```bash
# @command: [커맨드명]
# @purpose: [목적]
# @usage: [사용 시나리오]
# @keywords: [키워드] # 최소 3개
# @triggers: [트리거 패턴]
# @background: [true/false]
# @dependencies: [의존성]
# @output: [출력 형식]
```

**자동 통합**: 헤더 주석만 작성하면 유저룰스가 자동으로 검색 및 실행

### 3. 에이전트 팀 확장

#### 새 팀 추가

**절차**:

2. 슈퍼바이저 페르소나 정의
3. 필수 MCP 목록 정의
4. 유저룰스에 동적 생성 규칙 추가

**예시**:
```markdown
### 보안 팀 (Security Team)

**트리거**: 보안 관련 코드 변경, 의존성 업데이트
```

## 🔄 유지보수 전략

### 1. 정기 점검

#### 주간 점검

- `!AGENT_System/` 폴더 크기 확인
- LOG 파일 정리 (90일 이상 된 파일)
- 시스템 성능 확인

#### 월간 점검

- 모듈 업데이트 확인
- 커맨드 성능 분석
- 사용자 피드백 반영

### 2. 확장 계획

#### 단기 (1-3개월)

- 추가 에이전트 팀 (보안, 성능 등)
- 커맨드 확장 (특화 기능)
- MCP 통합 확대

#### 중기 (3-6개월)

- AI 모델 업데이트 대응
- 새로운 프로젝트 타입 지원
- 성능 최적화

#### 장기 (6개월+)

- 엔터프라이즈 기능 추가
- 멀티 프로젝트 관리
- 클라우드 통합

### 3. 문제 해결

#### 일반적인 문제

**문제**: 커맨드가 실행되지 않음
- **해결**: 헤더 주석 확인, 키워드 매칭 확인

**문제**: LOG가 생성되지 않음
- **해결**: 트리거 조건 확인, 파일 권한 확인

**문제**: 보고서 내용이 부정확함
- **해결**: MCP 활성화 확인, 검증 데이터 확인

#### 고급 문제

**문제**: 대량 파일 처리 시 성능 저하
- **해결**: 배치 처리, 백그라운드 실행 최적화

**문제**: 메모리 사용량 증가
- **해결**: LOG 파일 정리, 오래된 파일 아카이브
