# _COMMAND_LIBRARY 참조 문서 라이브러리

**역할**: 유저룰스에서 필요시 참조할 수 있는 상세 문서 및 실행 코드 템플릿

## 📋 폴더 구조

```
_COMMAND_LIBRARY/
├── mcp-command.md          # MCP 서버 트리거 매핑 참조 문서
├── user_command_guide.md   # 유저커맨드 생성 가이드 참조 문서
├── project-commands.md      # 실행 가능한 커맨드 코드 템플릿
├── project-init-guide.md   # 프로젝트 초기화 단계별 대화 가이드
├── agent-teams.md          # 프로젝트 에이전트 팀 생성 가이드
└── README.md               # 이 파일
```

## 🎯 각 파일의 역할

### 1. `mcp-command.md`
**역할**: MCP 서버 트리거 매핑 참조 문서
**사용 시점**: 
- 유저룰스에서 MCP 서버 매핑이 필요할 때
- 사용자 요청을 MCP 도구로 변환할 때
**내용**: 
- 각 MCP 서버별 명시적 트리거 및 의도 키워드
- MCP 도구 호출 예시
- 의도 파악 우선순위

### 2. `user_command_guide.md`
**역할**: 유저커맨드 생성 가이드 참조 문서
**사용 시점**:
- 새로운 커맨드를 생성할 때
- 커맨드 헤더 주석 형식을 확인할 때
**내용**:
- 커서 AI 유저 커맨드 구조
- 커맨드 정의 형식
- 헤더 주석 형식 (`@keywords` 등)
- 필수 원칙 및 예시

### 3. `project-commands.md`
**역할**: 실행 가능한 커맨드 코드 템플릿
**사용 시점**:
- 프로젝트 루트에 복사하여 실제 커맨드로 사용
- 유저룰스에서 `@keywords` 검색으로 커맨드 찾기
**내용**:
- 실제 실행 가능한 bash 함수들
- 헤더 주석이 포함된 커맨드들
- `cursor_command` 라우터 및 별칭

### 4. `project-init-guide.md`
**역할**: 프로젝트 초기화 단계별 대화 가이드
**사용 시점**:
- 프로젝트 초기화 요청 시 (1단계 대화 프로세스)
- "홈페이지 만들자" 같은 요청 시 즉시 파일 생성 방지
**내용**:
- 3단계 프로세스 (범주 파악 → 에이전트 팀 설정 → 세부 룰스 구축)
- 단계별 질문 목록
- 대화 원칙 및 프로세스 플로우

### 5. `agent-teams.md`
**역할**: 프로젝트 에이전트 팀 생성 가이드
**사용 시점**:
- 프로젝트 에이전트 팀 생성 시 (2단계)
- 프로젝트 타입별 팀 구성 템플릿 필요 시
**내용**:
- 이론적 기반 (Belbin, Adversarial Collaboration 등)
- 프로젝트 타입별 팀 템플릿
- 팀 역할 정의 및 논쟁 규칙

## 📋 유저룰스와의 관계

### 1. 컨텍스트 기반 검색
**프로세스**:
1. 사용자 요청/트리거 발생
2. 유저룰스에서 `@keywords` 검색
3. `project-commands.md`의 헤더 주석 읽기
4. 매칭 점수 계산 → 가장 적합한 커맨드 선택
5. 커맨드 실행

### 2. 참조 문서 활용
**프로세스**:
1. 유저룰스에서 상세 정보가 필요할 때
2. `_COMMAND_LIBRARY/` 폴더의 해당 문서 읽기
3. 상세 내용 확인 후 작업 수행

**예시**:
- MCP 트리거 매핑 필요 → `_COMMAND_LIBRARY/mcp-command.md` 읽기
- 커맨드 생성 필요 → `_COMMAND_LIBRARY/user_command_guide.md` 읽기
- 커맨드 검색 필요 → `project-commands.md`의 헤더 주석 검색
- 프로젝트 초기화 필요 → `_COMMAND_LIBRARY/project-init-guide.md` 읽기
- 에이전트 팀 생성 필요 → `_COMMAND_LIBRARY/agent-teams.md` 읽기
