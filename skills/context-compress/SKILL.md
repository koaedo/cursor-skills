# Context Compress Skill

컨텍스트 압축 - 토큰 절약 및 효율적 대화 관리

## When to Use
- 대화가 50턴 이상 진행 시
- 대화가 너무 길어져서 응답이 느려질 때
- 사용자가 "압축", "요약", "compress" 요청 시
- 완전히 새로운 주제로 전환할 때

---

## 🎯 목적
긴 대화에서 토큰을 절약하고 핵심 내용만 유지하여 효율적인 대화 진행

---

## 📋 압축 트리거

다음 상황에서 컨텍스트 압축을 **제안**합니다:

1. **대화 길이 기준**
   - 대화가 50턴 이상 진행 시
   - 대화가 너무 길어져서 응답이 느려질 때

2. **사용자 명시적 요청 시**
   - "압축", "요약", "compress"
   - "대화 정리", "토큰 절약"

3. **주제 전환 시**
   - 완전히 새로운 주제로 전환할 때
   - 이전 대화 내용이 더 이상 필요 없을 때

---

## 📋 압축 프로세스

```
압축 트리거 감지
    │
    ▼
현재 대화 분석
    │
    ├── 완료된 작업 추출
    ├── 진행 중인 작업 추출
    ├── 중요 결정사항 추출
    └── 핵심 컨텍스트 추출
    │
    ▼
압축 요약 생성
    │
    ▼
상세 내용은 !AGENT_System/에 LOG로 저장
    │
    ▼
압축된 컨텍스트로 대화 재시작
```

---

## 📁 압축 요약 형식

```markdown
## 🗜️ 컨텍스트 압축 요약

### 프로젝트 정보
- **프로젝트**: [프로젝트명]
- **압축 시각**: YYYY-MM-DD HH:MM

### 완료된 작업 (요약)
- [작업1 한 줄 요약]
- [작업2 한 줄 요약]

### 현재 상태
- **진행중**: [현재 작업]
- **다음 단계**: [예정 작업]

### 핵심 결정사항
- [결정1]
- [결정2]

### 주요 파일
- [파일1](경로)
- [파일2](경로)

---
*상세 내용: !AGENT_System/YYYYMMDD_HHMMSS_LOG_system_압축전대화.MD*
```

## 📋 압축 수준

| 수준 | 설명 | 사용 시점 |
|------|------|----------|
| **가벼운 압축** | 불필요한 대화만 제거 | 일반적인 경우 |
| **중간 압축** | 완료된 작업 요약 | 대화가 길어졌을 때 |
| **강한 압축** | 핵심만 유지 | 완전히 새로운 주제 시작 시 |

---

## ⚠️ 압축 시 유지해야 할 것

- ✅ 현재 진행 중인 작업 상태
- ✅ 중요한 결정사항
- ✅ 관련 파일 경로
- ✅ 에러/문제 상황 (해결 안 된 것)

## ❌ 압축 시 제거 가능한 것

- 완료된 단순 질문/답변
- 중복된 설명
- 탐색/검색 과정 상세 내용
- 이미 적용 완료된 코드 변경 과정
