---

## 📋 폴더 구조

```
commands/

├── project-commands.md         # 실행 가능한 커맨드 코드 템플릿

└── README.md                   # 이 파일
```

## 🎯 각 파일의 역할

### 1. `mcp-command.md`
**역할**: MCP 서버 트리거 매핑 참조 문서
**사용 시점**: 
- 유저룰스에서 MCP 서버 매핑이 필요할 때
- 사용자 요청을 MCP 도구로 변환할 때

### 2. `user_command_guide.md`
**역할**: 유저커맨드 생성 가이드 참조 문서
**사용 시점**:
- 새로운 커맨드를 생성할 때
- 커맨드 헤더 주석 형식을 확인할 때

### 3. `project-commands.md`
**역할**: 실행 가능한 커맨드 코드 템플릿
**사용 시점**:
- 프로젝트 루트에 복사하여 실제 커맨드로 사용
- 유저룰스에서 `@keywords` 검색으로 커맨드 찾기

### 4. `project-init-integration-plan.md`
**역할**: 프로젝트 초기화 3단계 연동 계획
**사용 시점**:
- 프로젝트 초기화 요청 시 (1단계 대화 프로세스)
- "홈페이지 만들자" 같은 요청 시

### 5. `project-rules-templates.md`
**역할**: 프로젝트룰스 파일 템플릿
**사용 시점**:
- AGENT_Teams.mdc, PROJECT_SETTINGS.mdc 등 생성 시

### 6. `agent-teams.md`
**역할**: 프로젝트 에이전트 팀 생성 가이드
**사용 시점**:
- 프로젝트 에이전트 팀 생성 시 (2단계)
- 프로젝트 타입별 팀 구성 템플릿 필요 시

### 7. `checkpoint-commands.md`
**역할**: 체크포인트 시스템 커맨드
**사용 시점**:
- 체크포인트 저장/복원 시

## 📋 유저룰스와의 관계

### 컨텍스트 기반 검색
**프로세스**:
1. 사용자 요청/트리거 발생
2. 유저룰스에서 `@keywords` 검색
3. `project-commands.md`의 헤더 주석 읽기
4. 매칭 점수 계산 → 가장 적합한 커맨드 선택
5. 커맨드 실행

### 참조 문서 활용
**프로세스**:
1. 유저룰스에서 상세 정보가 필요할 때
2. `commands/` 폴더의 해당 문서 읽기
3. 상세 내용 확인 후 작업 수행
